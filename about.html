About the Micro-Credentialing Project

A research initiative exploring blockchain-based verification of programming competencies in engineering education.

🎯 Project Vision

Traditional grading systems reduce student work to a single number — a GPA, a percentage, a letter grade. While grades matter for academic progression, they do not fully capture what students are capable of building or solving. In industry, what ultimately matters is demonstrable skill.

The Micro-Credentialing Project represents a paradigm shift in how we assess and document technical competencies in engineering education. Our goal is to move toward a credentialing model where specific programming competencies are assessed in structured, interview-style formats and, once mastered, recorded as verified credentials.

These credentials are designed to represent concrete, industry-relevant skills rather than simply course completion. Over time, students build a portfolio of verified competencies that extends beyond any single course — demonstrating not just that they took Embedded Systems, but that they passed structured assessments in specific technical skills under proctored conditions.

🔗 Why Blockchain Technology?

We use blockchain technology as the infrastructure for credential issuance. Blockchain provides an immutable, tamper-resistant ledger — meaning once a credential is issued, it cannot be altered or fabricated. It becomes a permanent, independently verifiable record of demonstrated skill.

In many ways, these credentials function similarly to non-fungible digital assets (NFTs), in that each one is unique and tied specifically to the student and the competency they demonstrated. However, unlike speculative NFTs associated with digital art or cryptocurrency markets, these credentials represent verified academic achievement and technical proficiency. Their value is not market-driven — it is skill-driven.

Key Benefits of Blockchain Credentials:

  • Immutable: Cannot be altered, forged, or deleted once issued
  • Verifiable: Employers can independently verify authenticity without contacting the university
  • Portable: Credentials belong to students and travel with them throughout their career
  • Permanent: Remain accessible regardless of institutional changes or system migrations
  • Granular: Document specific skills rather than broad course completion

🔬 Research Context

This pilot program is part of a PhD dissertation research project investigating the technical feasibility and educational value of blockchain-based micro-credentialing systems for engineering education. The research explores:

  • Integration of web-based coding assessments with automated proctoring systems
  • Privacy-aware biometric interfaces for identity verification
  • Blockchain technology for verifiable credential issuance
  • Human-in-the-loop approaches to academic integrity verification
  • Workforce development applications for industry-recognized credentials

Your participation as a student in this pilot directly contributes to understanding how these systems can be effectively implemented in real educational contexts.

⚙️ Platform Technology

The micro-credentialing platform is built on modern, scalable cloud infrastructure:

Core Technologies:

  • Frontend: React with TypeScript, Monaco Editor for code editing
  • Backend: AWS serverless architecture (Lambda, API Gateway, DynamoDB)
  • Code Execution: Judge0 API for automated testing and grading
  • Authentication: AWS Cognito for secure user management
  • Proctoring: MediaRecorder API for webcam recording, AWS Textract for OCR-based screenshot analysis
  • Storage: Amazon S3 for proctoring recordings
  • Blockchain: BCdiploma for credential issuance and verification

Key Features:

  • Automated code execution against multiple test cases
  • Real-time proctoring with webcam and screen monitoring
  • OCR-based detection of forbidden domains (ChatGPT, Stack Overflow, etc.)
  • Human-in-the-loop review of proctoring data before credential approval
  • Blockchain-verified credential issuance
  • Role-based access control (student/admin interfaces)

⚖️ Academic Integrity & Privacy

Human-in-the-Loop Approach

Unlike fully automated systems, our platform uses a "human-in-the-loop" philosophy. While the system automatically captures proctoring data and detects potential violations, final credential approval requires manual review by course staff. This approach provides better academic integrity outcomes than fully automated systems, particularly in educational contexts where false positives could significantly impact student outcomes.

Privacy Protections

  • Proctoring recordings are accessible only to authorized course staff
  • Recordings are used solely for academic integrity verification
  • Data is stored securely on AWS infrastructure with encryption at rest
  • Students have transparency into what is being recorded and why
  • Recordings are not shared with third parties or used for other purposes

What Gets Monitored

  • Webcam: Continuous recording for identity verification
  • Screenshots: Periodic captures analyzed via OCR for forbidden content
  • Detected Domains: Automated flagging of AI assistants, search engines, collaboration tools

Important: The goal is not surveillance, but rather to maintain the integrity and value of earned credentials. Verified credentials only have meaning if they represent authentic demonstrated competency.

🚀 Long-term Vision

This semester's pilot is the first step in a broader initiative to transform how technical skills are assessed and documented in engineering education.

Future Directions:

  • Expanded Curriculum: Additional assessments covering data structures, algorithms, embedded systems, and real-time programming
  • Industry Recognition: Partnerships with employers who recognize blockchain credentials as verified skill documentation
  • K-12 Outreach: Extending the platform to introduce programming competencies at earlier educational stages
  • Rural Access: Programs to provide credentialing opportunities in underserved communities
  • Institutional Adoption: Scaling the system across multiple courses and departments at Virginia Tech
  • Advanced Privacy Mechanisms: Research into zero-knowledge proofs and privacy-preserving biometric verification

Your participation in this pilot helps validate the technical feasibility and educational value of this approach, contributing to research that may shape the future of technical credentialing in engineering education.
