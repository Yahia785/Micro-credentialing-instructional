🔧 Troubleshooting Guides

Camera/Microphone Permission Issues

Problem: Browser won't allow webcam or microphone access

Solution:

  1. Click the camera icon in your Chrome address bar
  2. Select "Always allow [platform URL] to access your camera and microphone"
  3. Refresh the page
  4. If still not working, check Chrome Settings → Privacy and Security → Site Settings → Camera/Microphone

Screen Sharing Issues

Problem: Screen sharing not working or shows wrong screen

Solution:

  1. When prompted, select "Entire Screen" (not "Chrome Tab" or "Window")
  2. If you have multiple monitors, select the screen where you'll be coding
  3. Grant permission and verify the correct screen is being shared
  4. If sharing stops, refresh the page and start over

Code Submission Errors

Problem: Submission fails or returns unexpected results

Solution:

  1. Test your code thoroughly on OnlineGDB first
  2. Ensure output matches exactly (including capitalization and spacing)
  3. Check for infinite loops or excessive execution time
  4. Verify you're using a supported language version
  5. If issue persists, contact support immediately

Platform Not Loading

Problem: Platform pages won't load or show errors

Solution:

  1. Verify you're using Google Chrome (not Firefox, Safari, or Edge)
  2. Clear browser cache and cookies
  3. Disable browser extensions temporarily
  4. Try incognito/private browsing mode
  5. Check your internet connection
  6. If still not working, submit a support ticket

Account/Login Issues

Problem: Can't create account or login fails

Solution:

  1. Verify you're using your @vt.edu email address
  2. Check spam folder for verification code email
  3. Ensure password meets requirements (8+ characters, uppercase, lowercase, number, symbol)
  4. Try resetting password if you forgot it
  5. Contact support if verification code doesn't arrive within 5 minutes

🔗 Understanding Blockchain Credentials

What are Blockchain Credentials?

Blockchain credentials are digital certificates stored on an immutable, decentralized ledger. Once issued, they cannot be altered or fabricated, providing permanent verification of your demonstrated skills.

How to View Your Credentials

After successfully completing an assessment and passing the proctoring review, you will receive:

  • A unique credential URL stored on the blockchain
  • A digital certificate that can be shared with employers
  • Permanent proof of competency verification

Access your credentials through the Credentials tab on the platform.

⚖️ Academic Integrity

Honor Code

All work submitted through the micro-credentialing platform must be your own. This includes:

  • Writing your own code without copying from external sources
  • Not using AI assistants (ChatGPT, Claude, GitHub Copilot) during proctored assessments
  • Not collaborating with others during individual assessments
  • Not accessing unauthorized resources during proctored sessions

Proctoring Guidelines

During proctored assessments:

  • Allowed: Course notes, textbooks, official language documentation
  • Allowed: OnlineGDB for testing (if explicitly permitted for that assessment)
  • Not Allowed: ChatGPT, Claude, Google search, Stack Overflow, Discord, messaging apps
  • Not Allowed: Collaboration with peers or external help

Your screen recordings will be reviewed. Violations will be reported to course staff and may result in disciplinary action.
