General Guidelines

📋 Submission Policy

Most assessments allow one submission only. Make sure to thoroughly test your code before submitting. Use the onlineGDB tool linked in each problem for unlimited practice runs.

🎥 Proctoring Requirements

All assessments require webcam and screen sharing permissions. Recordings are reviewed only by course staff for academic integrity purposes. Ensure you have a quiet, well-lit environment and stable internet connection.

⏰ Deadlines

All deadlines are firm. Late submissions are not accepted. If you experience technical issues, report them immediately through the support system.

🏆 Credentials

Successfully completing an assessment with a passing score (typically 100%) earns you a blockchain-verified credential. Credentials are issued after course staff review proctoring recordings and approve the submission.

🔧 Technical Issues

If you encounter any technical problems during an assessment:

  • Do not refresh or close the browser
  • Take a screenshot of the error message
  • Submit a support ticket immediately
  • Contact course staff if the issue prevents submission

📊 Grading & Feedback

The platform provides immediate automated feedback on code execution. Your submission will show:

  • Number of test cases passed
  • Execution time and memory usage
  • Error messages (if any)

Final credential approval occurs after manual review of proctoring data by course staff.

⚠️ Important Reminders

  • Use Google Chrome only - Other browsers may cause compatibility issues
  • Test before submitting - You typically have only one submission attempt
  • Enable permissions - Both webcam and screen sharing must be allowed
  • Report issues immediately - Don't wait until after the deadline
  • Academic integrity - All work must be your own; violations will be reported
